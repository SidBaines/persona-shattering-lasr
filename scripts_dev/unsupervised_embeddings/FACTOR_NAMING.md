# Factor naming: paper-display ↔ internal labels

The four factors recovered from the v7-pf3 $k=4$ oblimin fit on
Llama-3.1-8B-Instruct are referred to by **two parallel naming schemes**:

| Variance order | Internal short name (used in code, data, HF artifacts) | Paper display name (used in `paper/sections/`, `paper/appendices/`) |
|----------------|---------------------------------------------------------|----------------------------------------------------------------------|
| F0 (11.7%)     | `Initiative`                                            | **I**nitiative                                                        |
| F1 (10.2%)     | `Warmth`                                                | **T**one                                                              |
| F2 (9.6%)      | `Pedagogy`                                              | **D**idacticism                                                       |
| F3 (9.3%)      | `Hedging`                                               | **E**pistemic Caution                                                 |

**The paper-display names spell TIDE** (Tone, Initiative, Didacticism,
Epistemic Caution) — chosen as a memorable narrative anchor for the
paper. The internal short names predate this rename and survive in:

- The `LoraValidation.factor` field in `LORA_VALIDATIONS` (and the HF
  monorepo subdir paths it points at: `unsupervised/initiative/`,
  `unsupervised/warmth/`, …).
- All `<label>_summary.json` and `<label>_scores.npz` files on the
  monorepo (`factor_summary[i].factor` is e.g. `"F1_Pedagogy"`, where
  the `F<n>_` prefix is the *pre-variance-sort* index — see also
  `_short_factor_name` in `lora_factor_shifts.py`).
- The Python factor canonical_factor_order constants in
  `scripts_dev/unsupervised_embeddings/analysis_for_paper.v2.py` and
  the LLM-labels JSON at
  `datasets/psychometric_fa_labels/analysis_for_paper.v2_k4/llama-3.1-8b/labeling/`.
- Scratch directories under `scratch/psychometric_fa_paper_v7pf3_k4/`.

## Translation rules

When you see one name and need the other:

- `Warmth` (data, code) ⇄ `Tone` (paper)
- `Pedagogy` (data, code) ⇄ `Didacticism` (paper)
- `Hedging` (data, code) ⇄ `Epistemic Caution` (paper)
- `Initiative` is the same in both schemes.

## Where to *not* rename

Do **not** rename the internal short names in code, HF subdirs, label
JSON files, or scratch outputs. Doing so would invalidate the
existing factor-identity anchors in `validate_lora.py`, the HF dataset
layout, and any cached scores that downstream LoRA-shift code already
hydrates by label. The paper-display rename is a presentation-layer
relabel only — applied via `PAPER_FACTOR_DISPLAY_NAMES` in
`analysis_for_paper.v2.py` when emitting paper figures.

## Where to do rename (paper)

- `paper/sections/unsupervised.tex` — factor list + main-text
  references.
- `paper/appendices/fa_factors.tex` — appendix per-factor item lists,
  validation captions, table column headers, residualized refit
  commentary, and any LoRA-shift narrative that names the factors.
- Figure axis labels generated by `analysis_for_paper.v2.py` (use the
  `PAPER_FACTOR_DISPLAY_NAMES` mapping).

If you find an older reference somewhere that still uses Warmth /
Pedagogy / Hedging in *paper* prose, treat it as a stale artifact
and update it. Code, scratch, and HF references should stay.
